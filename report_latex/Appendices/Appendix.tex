\chapter{Supplementary Sweep Results}

This appendix provides additional hyperparameter sweep visualisations and configuration details that complement the main results presented in the Implementation chapter.

\section{Sweep Configuration Space}

The hyperparameter sweep explored a grid of $n_t \in \{1, 3, 5, 7, 9\}$ (time levels for flow discretisation) and $n_{\text{noise}} \in \{1, 3, 5, 7, 9, 11, 13, 15\}$ (noise-augmented training duplicates), yielding $5 \times 8 = 40$ evaluated configurations. All runs used the HS3F backbone with default XGBoost parameters (100 estimators, max depth 6, learning rate 0.1) and 50{,}000 autoregressive training rows. Synthetic sample size was matched to the evaluation split for TSTR assessment.

\section{XGBoost Configuration}

All XGBoost models (both flow-matching regressors and categorical classifiers) used the following default parameters throughout the sweep:

\code{learning\_rate} & 0.1 \\
\code{tree\_method} & \code{hist} \\
\code{n\_jobs} & $-1$ (all cores) \\
\code{random\_state} & 42 \\
\bottomrule
\end{tabular}
\caption{Default XGBoost hyperparameters used in all sweep configurations.}

These parameters were not tuned as part of the sweep; they represent a single fixed operating point. Tuning XGBoost hyperparameters jointly with $n_t$ and $n_{\text{noise}}$ is a natural extension but would expand the search space multiplicatively.
